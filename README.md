# RISC-V Pipelined Core -- Full Open-Source ASIC Flow

A 4-stage pipelined RV32I subset core (IF -> EX -> MEM -> WB), carried
through a complete open-source ASIC flow -- Yosys synthesis, OpenROAD
place-and-route, Sky130 PDK -- to signed-off GDSII.

## Why this project

Most student RTL projects stop at simulation. This one goes all the way to
a manufacturable layout with real timing/power/area numbers, closing the
loop between "the RTL is correct" and "the RTL actually becomes a chip."

## Pipeline design

See `docs/ARCHITECTURE.md` for the full writeup. Summary:

- **ISA subset**: ADD/SUB/AND/OR/XOR/SLL/SRL/SLT (R and I-type), LW, SW,
  BEQ, BNE, JAL.
- **Hazards handled**:
  - RAW (ALU producer -> consumer): resolved via EX/MEM and MEM/WB
    forwarding into the EX-stage ALU operand muxes.
  - Load-use hazard: detected and resolved with a 1-cycle stall (the
    loaded value isn't ready in time for same-cycle forwarding).
  - Control hazard (branch/JAL): resolved in EX; the wrong-path
    instruction already in IF is squashed on a taken branch/jump.

## Verification

`tb/tb_riscv_core.v` loads a test program (`sim/program.hex`, generated by
`sim/assembler.py` -- a small assembler for this core's ISA subset) and
checks every written register against `sim/golden_regs.hex`, computed by
`sim/golden_model.py` -- a from-scratch RV32I interpreter that decodes and
executes the same machine code independently of the RTL or the assembler's
own logic. The test program exercises every hazard path above: ALU
forwarding, the load-use stall, a taken branch, a not-taken branch, and
JAL. Result: bit-exact match on every register the program writes.

```bash
iverilog -o sim.out rtl/regfile.v rtl/alu.v rtl/imem.v rtl/dmem.v rtl/riscv_core.v tb/tb_riscv_core.v
vvp sim.out
```

## ASIC flow results (Yosys + OpenROAD, Sky130 PDK)

Full details and the honest optimization story (including a stretch
target that failed timing) in `openlane/riscv_core/RESULTS.md`. Headline
numbers:

| | 100 MHz baseline | 333 MHz optimized |
|---|---|---|
| Clock period | 10 ns | 3 ns |
| Timing (WNS/TNS) | 0.0 / 0.0 (met) | 0.0 / 0.0 (met) |
| Critical path | 1.57 ns | 1.87 ns |
| DRC / LVS / routing violations | 0 / 0 / 0 | 0 / 0 / 0 |
| Core area | 613,701 um^2 | 613,701 um^2 |
| Power (typical) | ~0.75 uW | ~2.79 uW |

Both operating points pass clean signoff (Magic DRC, netgen LVS, OpenROAD
STA) with zero violations.
